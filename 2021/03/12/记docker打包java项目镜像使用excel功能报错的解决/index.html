记docker打包java项目镜像使用excel功能报错的解决

一般使用docker打java程序镜像会用openjdk-alpine。这个是最小化的java环境，就像以前装linux虚拟机，都是用最小化安装系统，系统能跑起来就行，要用什么再装什么，没必要去装个4g的完整版，浪费时间，还占用资源。openjdk-alpine这个镜像就是同理，我们用的是springboot和esay execl做了导入导出excel的接口，在打包的时候就需要安装字符集（代码如下），否则调用接口时就会报错。

优雅的方式

我们是用jenkens做了cicd的自动化，每次程序mr提交后都会重新打包构建镜像，然后就发现由于安装字符集需要每次下载字符集并安装到镜像内，会由于网络原因造成jenkins自动部署失败，所以可以用openjdk-alpine为基础镜像，制作了包含字符集的openjdk镜像，如果有其他的需要打进镜像的程序等也可以同理处理之。
